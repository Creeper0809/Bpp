# Contributing

이 페이지는 Bpp 프로젝트에 기여할 때의 기준과 절차를 설명합니다.

## 1. 기여 기본 원칙

- 기능 추가 시 테스트 동시 추가
- 실패 케이스는 진단 메시지까지 검증
- 리팩터링은 동작 보존을 최우선

## 2. 작업 절차

1. 이슈 또는 개선 목표 정의
2. 브랜치 생성
3. 코드 수정 + 테스트 추가/수정
4. 로컬 빌드/테스트 통과 확인
5. PR 제출(변경 이유/영향/검증 결과 포함)

## 3. 코드 변경 가이드

- 의미 없는 대규모 포맷 변경 지양
- 패스 단위 변경 시 책임 범위를 명확히 유지
- 파서/컴파일러/코드젠 경계를 흐리지 않기

## 4. 리뷰 체크리스트

- 기존 동작 회귀 여부
- 오류 처리/메시지 품질
- 플랫폼 영향(Linux/Windows)
- 테스트 커버리지 충분성

## 5. 문서 기여

- 위키 페이지 제목은 영어
- 본문 설명은 한글
- Home에는 요약과 링크만, 상세는 하위 페이지로 분리
